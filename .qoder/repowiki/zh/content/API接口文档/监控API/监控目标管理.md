## 简介
本文件为ZBH2平台监控目标管理API的详细接口文档。文档覆盖监控目标的完整CRUD操作接口，包括目标类型定义、主机配置、端口设置、状态管理等功能。详细解释监控目标的健康状态检查机制、在线离线状态切换逻辑，以及目标配置参数的验证规则、默认值设置和错误处理策略。同时提供完整的请求/响应示例，涵盖目标创建、更新、删除、状态查询等场景，并给出监控目标分类管理、批量操作和状态监控的最佳实践建议。

#### 监控目标字段定义
- **id**: 自增主键，唯一标识监控目标
- **name**: 监控目标名称，必填字段
- **type**: 监控目标类型，枚举值包括device、system、database、service
- **host**: 主机地址，可选字段
- **port**: 端口号，可选字段
- **description**: 目标描述，可选字段
- **status**: 当前状态，枚举值包括online、offline、warning、critical，默认online
- **config**: 配置JSON对象，可选字段
- **createdAt/updatedAt**: 创建和更新时间戳

#### 默认值和约束
- 类型字段默认值为'system'
- 状态字段默认值为'online'
- 收集间隔默认值为60秒
- 启用状态默认值为1（启用）

```mermaid
CheckRole --> |管理员| Proceed[允许访问]
Unauthorized --> End[结束]
Forbidden --> End
Proceed --> End
```

### 健康状态检查机制

监控目标的健康状态检查机制通过以下方式实现：

```mermaid
flowchart TD
Start[开始监控检查] --> GetTargets[获取所有监控目标]
GetTargets --> CheckType{检查目标类型}
CheckType --> |服务器| CheckServer[检查服务器状态]
CheckType --> |网络设备| CheckNetwork[检查网络设备状态]
CheckType --> |数据库| CheckDatabase[检查数据库状态]
CheckType --> |应用服务| CheckService[检查应用服务状态]
CheckServer --> PingHost[Ping主机连通性]
CheckNetwork --> SNMPCheck[SNMP查询]
CheckDatabase --> SQLTest[SQL连接测试]
CheckService --> HTTPCheck[HTTP请求测试]
PingHost --> HostResult{检查结果}
SNMPCheck --> NetResult{检查结果}
SQLTest --> DBResult{检查结果}
HTTPCheck --> ServiceResult{检查结果}
HostResult --> |正常| SetOnline[设置为在线]
HostResult --> |异常| SetOffline[设置为离线]
NetResult --> |正常| SetOnline
NetResult --> |异常| SetOffline
DBResult --> |正常| SetOnline
DBResult --> |异常| SetOffline
ServiceResult --> |正常| SetOnline
ServiceResult --> |异常| SetOffline
SetOnline --> UpdateStatus[更新状态到数据库]
SetOffline --> UpdateStatus
UpdateStatus --> End[完成检查]
```

#### 状态切换逻辑
监控目标的状态切换遵循以下规则：
1. **在线状态**：目标能够正常响应检查请求
2. **离线状态**：目标无法响应检查请求或连接失败
3. **警告状态**：目标部分功能异常但未完全失效
4. **严重状态**：目标完全不可用或出现严重问题

#### 状态检查频率
- 默认检查间隔：60秒
- 可根据目标类型调整检查频率
- 关键目标可设置更短的检查间隔

### 常见错误及解决方案

#### 认证相关错误
- **401 未授权**: 检查会话Cookie是否正确传递，确认用户已登录
- **403 权限不足**: 确认用户角色为admin，检查用户权限设置

#### 数据验证错误
- **400 参数错误**: 检查请求参数格式，确保必填字段完整
- **404 目标不存在**: 确认目标ID有效，检查数据库中是否存在该记录

#### 数据库操作错误
- **500 服务器内部错误**: 检查数据库连接状态，查看数据库日志
- **并发冲突**: 实现重试机制，使用数据库事务处理

#### 前端集成问题
- **跨域问题**: 检查CORS配置，确保API域名正确设置
- **会话丢失**: 确认Cookie配置，检查浏览器隐私设置

## 简介
ZBH2 是一个基于 Node.js + React + SQLite 的正版化软件管理 B/S 平台，提供软件分类下载、帮助文档浏览、Windows/Office/WPS 激活码发放，以及完整的管理后台。本指南将帮助你在最短时间内完成环境准备、安装与启动，并了解基本功能与安全注意事项。

## 环境要求
- Node.js 版本要求：18 或更高
- 包管理器：pnpm 8 或更高
- 其他：SQLite 数据库文件与上传目录将由应用自动创建（无需手动初始化）

## 项目结构概览
- apps/server：Fastify 后端 API，包含数据库迁移、种子、中间件与路由
- apps/web：React + Vite 前端，包含页面、布局、API 客户端与主题
- packages/shared：前后端共享的 Zod Schema 与类型定义
- tools/ActivationClientWpf：Windows 平台的激活客户端演示（WPF）
- data/：SQLite 数据库文件与上传文件（已忽略在版本控制之外）

## 常见问题排查
- Node.js 或 pnpm 版本不满足要求
  - 现象：安装阶段报错或无法启动
  - 解决：升级 Node.js 至 18+，pnpm 至 8+
  - 参考：[README.md:7-11](file://README.md#L7-L11)，[package.json:13-15](file://package.json#L13-L15)

- 数据库迁移失败或找不到表
  - 现象：启动后端时报错或访问接口异常
  - 解决：确保已执行数据库迁移与种子脚本
  - 参考：[README.md:18-20](file://README.md#L18-L20)，[apps/server/src/db/migrate.ts:14-15](file://apps/server/src/db/migrate.ts#L14-L15)，[apps/server/src/db/seed.ts:94-95](file://apps/server/src/db/seed.ts#L94-L95)

- 前端无法访问 /api 接口
  - 现象：浏览器 Network 面板显示 404 或跨域错误
  - 解决：确认前端开发服务器代理已生效；后端已启动且监听 7500 端口
  - 参考：[apps/web/vite.config.ts:8-10](file://apps/web/vite.config.ts#L8-L10)，[apps/server/src/index.ts:51-53](file://apps/server/src/index.ts#L51-L53)

- 登录失败或提示权限不足
  - 现象：登录返回错误或访问管理后台 403
  - 解决：确认使用默认管理员账号 admin/admin123；检查会话是否正确写入 Cookie；确认用户状态为 active
  - 参考：[apps/server/src/routes/auth.ts:14-32](file://apps/server/src/routes/auth.ts#L14-L32)，[apps/server/src/middleware/auth.ts:35-39](file://apps/server/src/middleware/auth.ts#L35-L39)

- 上传文件无法访问
  - 现象：上传成功但无法通过 /uploads/ 访问
  - 解决：确认 uploads 目录存在且后端已注册静态资源插件
  - 参考：[apps/server/src/index.ts:24-35](file://apps/server/src/index.ts#L24-L35)

- 端口占用冲突
  - 现象：启动失败并提示端口已被占用
  - 解决：修改后端 PORT 环境变量或释放端口
  - 参考：[apps/server/src/index.ts:51-52](file://apps/server/src/index.ts#L51-L52)，[README.md:99-102](file://README.md#L99-L102)
